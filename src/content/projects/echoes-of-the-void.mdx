# Visão Geral do Projeto

Este projeto foi desenvolvido durante uma game jam de 48 horas e posteriormente expandido. O objetivo principal era criar um **sistema de geração procedural** de inimigos que se adaptasse dinamicamente ao desempenho do jogador.

## O Desafio Técnico

O maior desafio foi a otimização de performance. Com centenas de projéteis e inimigos na tela, a arquitetura tradicional do Unity era lenta. Para resolver isso, utilizamos:

1.  **DOTS (ECS):** Para os sistemas de *movement* e *collision* de projéteis.
2.  **Job System:** Para o cálculo de *pathfinding* de inimigos complexos.

## Aprendizados

Aprender a balancear a dificuldade e garantir que a IA dos inimigos fosse envolvente foi crucial. O uso do padrão de projeto Observer para a comunicação entre o sistema de *spawning* e o *score* do jogador tornou a manutenção do código muito mais limpa.
